# Linuxの構造

- Linuxでは、FHS（Filesystem Hierarchy Standard）という規定に基づいて標準化されたファイルシステム（ディレクトリ構造）を採用している。
<br>これにより、CentOSやDebianなど異なるディストリビューションでも基本的なディレクトリ構造は統一されている。

- FHSは、ディレクトリの最上位階層であるルートファイルシステム（/） に含める必要があるディレクトリや各ディレクトリの役割、ファイルの配置場所を定義していて、ルートには以下の5つのファイルを含める必要がある。

<br>

## ルートに含める必要があるディレクトリ**
| ディレクトリ名 | 説明 |
| --- | --- |
| /bin | 一般ユーザーでも実行可能な基本的なコマンドを格納 |
| /sbin | rootユーザーのみが実行可能なシステム管理コマンドを格納 |
| /etc | システムやコマンドの各種設定ファイル、スクリプトファイルを格納 |
| /dev | ハードディスク、CD-ROM、DVD-ROMなどデバイスファイルを格納 |
| /lib | 共有ライブラリ、カーネルモジュールを格納 |

## ルート以外の主なディレクトリ
| ディレクトリ名 | 説明 |
| --- | --- |
| /opt | 追加のパッケージやプログラムがインストールされるディレクトリ |
| /root | rootユーザー専用のディレクトリ |
| /home | 一般ユーザー専用のディレクトリ |
| /tmp | 一時的な作業ディレクトリ、全てのユーザーが読み書き可能 |
| /var ※1 | システム関連のログファイルを格納 |
| /usr ※2 | 一般ユーザーが共有するコマンド、ライブラリを格納 |

## var配下の主なディレクトリ ※1
| ディレクトリ名 | 説明 |
| --- | --- |
| /var/log | システムのログファイル（messages）、メールログ（maillog）などを格納 |
| /var/run | システム状態を示すファイルを格納 |
| /var/spool | 送信待ちメール（/var/spool/mqueue）などを格納 |

- 補足：cronの実態はどこにあるか
  - ユーザーごとに存在する定時処理を行うための`cron`の実体の格納先は、<br>`/var/spool/cron`に格納されている。

## var配下の主なディレクトリ ※2
| ディレクトリ名 | 説明 |
| --- | --- |
| /usr/bin | 一般ユーザー、rootユーザーが使用する基本コマンドを格納 |
| /usr/sbin | システム管理コマンドを格納 |
| /usr/lib | プログロムに必要な共有ライブラリを格納 |
